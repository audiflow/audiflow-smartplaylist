SmartPlaylist Definition

Type: object

Defines how a smart playlist groups, filters, sorts, and displays episodes for a specific podcast. Each definition file lives at {patternId}/playlists/{id}.json. Multiple definitions under the same pattern share the podcast's episode pool and claim episodes through a priority-based system.

Type: const

Groups episodes by matching their titles against regex patterns defined in the groups array. Groups are evaluated in order; the first matching pattern claims each episode. root priority
Type: integer Default: 0

Controls episode claiming order among sibling definitions within the same pattern. Processing order: (1) definitions with at least one filter are processed first, sorted by priority ascending; (2) definitions without any filters (fallbacks) are processed second, also sorted by priority ascending. Within each group, lower priority numbers claim matching episodes first. Once a filtered definition claims episodes, those episodes are unavailable to subsequent definitions.

Value must be greater or equal to 0

root groups
Type: array

Static group definitions, primarily used by the category resolver. Each group defines a regex pattern for matching episode titles. Groups are evaluated in order; the first matching pattern claims each episode (first-match-wins). A group without a pattern acts as a catch-all for unmatched episodes and should be placed last. For the titleAppearanceOrder resolver, only groups[0].pattern is used as a fallback extraction pattern when titleExtractor is not provided. Groups can selectively override display, episodeList, and episodeExtractor defaults.
