SmartPlaylist Definition


Defines how a playlist organizes, filters, sorts, and displays episodes for a podcast. Each definition file lives at {patternId}/playlists/{id}.json. Multiple definitions under the same config share the podcast's episode pool; definitions with filters run first, then filter-less fallbacks claim remaining episodes.

Type: const

Groups episodes by their season number from feed data or titles. Season numbers come from the numbering rule (parsing titles/descriptions) if configured, falling back to RSS metadata (itunes:season). Each distinct season becomes a separate group. Episodes without a season number are left ungrouped. Best for podcasts that tag seasons or use consistent title patterns like 'S01E01'. Supports: titleExtractor, numberingExtractor.

root presentation oneOf item 0
Type: const

Each resolver result becomes its own playlist, selectable from a dropdown. One definition produces many playlists, each containing episodes that share the same grouping key. Tapping a playlist shows its episodes directly. Best when each group (e.g., a season or year) should stand alone as an independent playlist.

Specific value: "separate"
root presentation oneOf item 1
Type: const

All resolver results are collected as group cards inside a single playlist. One definition produces one playlist with sub-groups. Tapping the playlist shows group cards, and tapping a group card navigates to its episodes. Enables groupList settings (yearBinding, userSortable, showDateRange, sort). Best when groups should be browsed together under a unified playlist.

Specific value: "combined"

Type: object

Generates display names from episode data for group titles. Used by seasonNumber, year, and titleDiscovery resolvers. Not used with titleClassifier, which uses the display names you define directly. Runs against a representative episode from each group (typically the first).

Which part of the episode to read: 'title' (episode title), 'description' (episode description), 'seasonNumber' (season number from feed or numbering rule), or 'episodeNumber' (episode number from feed or numbering rule).

root titleExtractor pattern
Type: string

A text pattern to find in the source value. Use parentheses () to mark the part you want to extract. When omitted, the raw source value is used directly (useful with a template like 'Season {value}').

root titleExtractor fallbackValue
Type: string

A default name used when the source is 'seasonNumber' or 'episodeNumber' and the value is missing or zero. Checked before pattern matching. Has no effect for 'title' or 'description' sources. Useful for labeling specials (e.g., 'Extras' for season 0).

root numberingExtractor
Type: object

Finds season and episode numbers in episode titles or descriptions. Overrides feed data (itunes:season, itunes:episode) with values parsed from episode content. Essential for podcasts that encode numbering in titles (e.g., '[62-15] Title') rather than using standard RSS tags. Can be customized per-group via GroupDef.numberingExtractor.

root groups
Type: array

Group definitions, primarily used by titleClassifier. Each group defines a text pattern for matching episode titles. Groups are evaluated in order; the first match claims each episode. A group without a pattern acts as a catch-all for unmatched episodes and should be placed last. For titleDiscovery, only groups[0].pattern is used as a fallback when titleExtractor is not provided. Groups can customize display, episodeList, and numberingExtractor settings.

 No Additional Items

root groups GroupDef
Type: object

A group definition. Groups with a pattern match episodes by title; a group without a pattern serves as a catch-all for unmatched episodes. Groups are evaluated in order (first match wins). At most one catch-all should exist and should be placed last. Groups can customize display, episodeList, and numberingExtractor settings.
