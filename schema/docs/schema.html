Type: object

Defines how a smart playlist groups, filters, sorts, and displays episodes for a specific podcast. Each definition file lives at {patternId}/playlists/{id}.json. Multiple definitions under the same pattern share the podcast's episode pool and claim episodes through a priority-based system.

root priority
Type: integer Default: 0

Controls episode claiming order among sibling definitions within the same pattern. Processing order: (1) definitions with at least one filter (titleFilter, excludeFilter, or requireFilter) are processed first, sorted by priority ascending; (2) definitions without any filters (fallbacks) are processed second, also sorted by priority ascending. Within each group, lower priority numbers claim matching episodes first. Once a filtered definition claims episodes, those episodes are unavailable to subsequent definitions.

Value must be greater or equal to 0

root contentType
Type: enum (of string)

Determines how resolver results are presented in the UI. When omitted, defaults to 'episodes'. In 'episodes' mode, each resolver result (season, year, category match) becomes a separate top-level smart playlist; tapping shows episodes directly. In 'groups' mode, all resolver results are collected as groups inside a single parent playlist; tapping the playlist shows group cards, and tapping a group card navigates to its episodes. The groups mode enables yearHeaderMode, showSeasonNumber, showDateRange, and customSort features.

Must be one of:

  • "episodes"
  • "groups"

root yearHeaderMode
Type: enum (of string)

Controls how groups are organized under year headers in the group list view. Only meaningful when contentType is 'groups'. When omitted, defaults to 'none'. 'none': groups are displayed in a flat list without year headers. 'firstEpisode': each group appears once, placed under the year of its earliest episode; groups are organized under sticky year headers. 'perEpisode': a group appears multiple times, once for each year it contains episodes; tapping a group-year entry shows only that year's episodes from the group. When yearHeaderMode is not 'none', the sort order toggle is automatically shown in the UI.

Must be one of:

root titleFilter
Type: string

Case-insensitive regex inclusion filter applied to episode titles before resolver processing. Only episodes whose title matches this pattern are passed to the resolver. Having any filter (titleFilter, excludeFilter, or requireFilter) makes this definition a 'filtered' definition, which is processed before filter-less fallback definitions. Matched episodes are claimed and unavailable to later definitions.

root excludeFilter
Type: string

Case-insensitive regex exclusion filter applied to episode titles. Episodes whose title matches this pattern are excluded, even if they passed titleFilter. Applied after titleFilter. Useful for carving out subsets: e.g., one definition uses titleFilter to match all numbered episodes, while excludeFilter removes a specific sub-series that another definition will claim.

root requireFilter
Type: string

Case-insensitive regex requirement filter applied to episode titles. Episodes must match this pattern to be included. Applied after titleFilter and excludeFilter. Semantically similar to titleFilter but intended for narrowing within an already-filtered set: e.g., titleFilter selects numbered episodes, requireFilter further restricts to a specific sub-series.

Group key assigned to episodes that have a null or zero seasonNumber in RSS metadata. Used only by the rss resolver. When set, episodes without a valid season number are placed into a group with this key (e.g., 0 for a 'Specials' group).
